## 这是什么

这是一个类似“运行在微信里的智能体”的 AI 助手。它可以接入 DeepSeek、OpenAI 等大模型，也可以连接你自定义的 MCP 服务，并通过自定义提示词塑造专属的说话风格和处理规则。

它可以帮助你管理自动回复、查询微信里的聊天和联系人信息、整理数据、生成文件、调用外部工具，并在你明确要求时帮你发送消息。

它的目标不是替代你聊天，而是让你拥有一个可以理解微信上下文、能处理微信数据、能调用工具执行任务的个人智能体。

## 你可以用它做什么

### 1. AI 聊天助手

你可以打开“微信 Agent”和 AI 对话，直接用自然语言提出需求。

例如：

- “帮我查一下最近有哪些群聊比较活跃”
- “帮我找一下没有备注名的好友”
- “帮我把查询结果导出成表格”
- “帮我总结一下某个群最近在聊什么”
- “帮我查一下这个联系人最近的聊天记录”

AI 会根据你的问题选择合适的能力执行任务，并把结果展示给你。

### 2. 自动回复

你可以开启全局自动回复，让 AI 在收到消息时自动帮你回复。

自动回复支持：

- 私聊自动回复
- 群聊自动回复
- 每个群单独配置
- 当前会话临时关闭自动回复
- 调整 AI 回复风格
- 设置读取最近历史消息数量
- 开启“单条消息必回”策略

群聊里默认只在别人 @ 你时才回复，避免 AI 在群里随意插话。

### 3. 个性化人设和提示词

你可以配置 AI 的说话风格，让它更像你想要的样子。

支持：

- 使用预设性格
- 新增自定义提示词
- 开启或关闭某条提示词
- 预览提示词内容
- 使用 AI 优化你写的提示词
- 使用内置的聊天表达技巧，让回复更像真人说话

这些配置会影响 AI 的整体表达方式。

你也可以把提示词理解成智能体的“行为说明书”：它决定 AI 应该用什么身份、语气、边界和方式来处理你的微信任务。

### 4. 查询联系人和好友关系

AI 可以帮你整理微信联系人。

例如：

- 查询全部双向好友
- 查询我关注的人
- 查询关注我但我未关注的人
- 查询黑名单联系人
- 查询已删除或异常状态联系人
- 查询没有备注名的好友
- 查询重复备注名
- 按昵称、备注、微信号搜索联系人

你可以直接问：

“帮我列出所有双向好友，并导出成表格。”

### 5. 查询群聊信息

AI 可以帮你查看和整理群聊。

例如：

- 查询我有哪些群聊
- 查询某个群的成员
- 查询群聊活跃度
- 查询某个群最近的聊天记录
- 总结某个群最近在聊什么

你可以直接问：

“帮我看看最近 7 天最活跃的群有哪些。”

### 6. 查询聊天记录

AI 可以读取微信里的历史聊天内容，用来帮你检索、整理和总结。

例如：

- 查询某个联系人最近的聊天记录
- 搜索包含某个关键词的消息
- 查询图片、语音、文件、引用消息等类型记录
- 对聊天内容做总结
- 根据聊天记录生成报告

语音消息如果已经有转文字缓存，也可以作为文本加入分析。

### 7. 导出表格和文件

当查询结果很多时，AI 可以帮你保存成文件。

支持：

- 导出 CSV 表格
- 写入文本文件
- 写入 Markdown 文件
- 查看文件路径
- 读取已有文本文件
- 列出目录内容

例如：

“帮我把所有没有备注名的好友导出成表格。”

完成后 AI 会告诉你文件保存位置。

### 8. 发送微信消息

当你明确要求发送时，AI 可以帮你发送微信消息。

支持：

- 发送文本消息
- 在群里 @ 指定成员
- 引用某条消息回复
- 发送本地图片
- 发送本地文件
- 发送联系人名片

为了避免误操作，涉及真实发送消息时，你需要明确告诉 AI 发给谁、发什么。

例如：

“把这个表格发给文件传输助手。”

### 9. 外部工具扩展

你可以配置自定义 MCP 服务，让 AI 获得更多外部工具能力。

例如：

- 联网搜索
- 调用第三方服务
- 读取远程资料
- 使用你自己部署的工具服务

如果你配置了自定义 MCP 服务，AI 启动时会尝试连接，并在可用时自动使用。

### 10. 模型配置

你可以根据自己的使用习惯配置 AI 模型。

支持：

- DeepSeek
- OpenAI
- 自定义服务地址
- 自定义 API Key

配置完成后，微信 Agent 和自动回复都会使用你选择的模型能力。

### 11. 调试和查看提示词

调试模式下可以查看 AI 当前使用的提示词内容，方便你判断 AI 为什么会这样回答。

你也可以查看一些测试界面，用来确认消息列表、展示样式和工具输出是否正常。

## 如何使用

### 打开主面板

在微信内点击插件入口，会直接打开 AI 控制面板。

你可以在这里：

- 开关自动回复
- 配置 AI 模型
- 配置回复风格
- 进入更多设置
- 打开微信 Agent
- 管理外部工具服务
- 管理提示词

### 使用微信 Agent

进入“微信 Agent”后，你会看到一个对话界面。

顶部会显示“我能帮你干什么？”，点击展开可以查看当前可用能力。

底部输入框用于输入任务，发送后 AI 会开始执行。右上或底部的“锁定输出”开关用于控制是否自动滚动到最新消息：

- 锁定开：AI 回复、工具调用和结果会自动滚动到最新位置。
- 锁定关：列表不会强制跳到底部，适合你边执行边查看前面的内容。

你可以像聊天一样输入任务，例如：

- “帮我查一下我微信有哪些群”
- “帮我找出没有备注的好友”
- “帮我导出全部双向好友”
- “帮我总结一下这个群最近的聊天”
- “帮我把结果保存成文件”

AI 会在执行过程中展示：

- 当前状态
- 工具调用
- 过程分析
- 最终结果

工具调用和工具结果会单独显示成卡片。工具参数、工具结果或最终回复很长时，界面会自动折叠，点击卡片可以展开或收起。

长按消息、工具调用或工具结果，可以复制完整内容。即使界面里只展示了折叠后的前半部分，复制时仍会复制完整文本。

### 配置自动回复

在 AI 控制面板中可以打开或关闭自动回复。

开启后，AI 会根据你的配置和聊天上下文生成回复。

如果你只想临时关闭某个联系人或某个群的自动回复，可以在当前会话设置里关闭。

### 配置提示词

你可以进入提示词设置，管理 AI 的行为规则和说话风格。

支持：

- 查看内置提示词
- 开启或关闭提示词
- 新增自定义提示词
- 编辑自定义提示词
- 删除自定义提示词
- 使用 AI 优化提示词

内置提示词可以关闭和预览，但不能删除。

### 配置外部工具

进入外部工具配置后，可以新增工具服务。

支持手动填写，也支持快速粘贴 JSON 配置。

如果你还没有 MCP 服务地址，可以点击“MCP 市场”打开 ModelScope MCP 市场：

https://modelscope.cn/mcp

在市场里选择需要的 MCP 服务后，将服务地址或配置内容复制回来，再在外部工具配置页里新增或粘贴配置。

新增 MCP 服务时需要填写：

- 服务名称：方便你识别这个服务。
- 服务地址：MCP 服务 URL。
- 连接方式：通常使用 Streamable HTTP；如果服务方要求 SSE，再切换到 SSE。
- Bearer Token：如果服务方要求鉴权，需要填写 token；没有要求可以留空。
- 是否启用：关闭后该服务不会被 Agent 加载。

配置完成并启用后，重新打开微信 Agent，AI 会自动加载可用工具。

如果 MCP 配置发生变化，建议关闭并重新打开微信 Agent，让新的工具列表重新加载。

## 使用建议

### 1. 任务越明确，结果越稳定

建议你告诉 AI：

- 要查什么
- 查哪个联系人或群
- 时间范围是多少
- 是否需要导出文件
- 是否需要发送给别人

例如：

“帮我查最近 30 天和张三的聊天记录，筛选包含‘合同’的消息，并导出成表格。”

### 2. 涉及发送消息时要说清楚

发送消息是真实操作。请明确说明：

- 发给谁
- 发什么内容
- 是否需要引用回复
- 是否需要 @ 某个人
- 是否发送文件或图片

### 3. 大量数据建议导出文件

如果查询结果很多，不建议让 AI 一次性全部显示在聊天窗口。

你可以让 AI：

- 导出成表格
- 保存成 Markdown
- 保存成文本文件
- 只展示摘要

### 4. 群聊自动回复要谨慎

群聊消息复杂，建议只在你确实需要时开启。

当前设计下，群聊默认只在别人 @ 你时回复，减少打扰。

### 5. 外部工具失效时重新配置

有些外部工具服务可能会过期、失效或无法连接。

如果 AI 提示连接失败，需要进入外部工具配置页更新地址或重新启用。

如果你是从 MCP 市场复制的服务地址，注意检查：

- 地址是否复制完整。
- 服务类型是否和市场说明一致。
- token 是否过期。
- 手机网络是否可以访问对应服务。

### 6. 长内容先折叠再查看

工具参数、工具结果和最终回复超过一定长度时会自动折叠，避免界面被超长文本撑满。

你可以：

- 点击卡片展开完整内容。
- 再次点击收起。
- 长按复制完整内容。
- 让 AI 把大量结果保存成文件后再查看。

## 注意事项

- AI 可能会理解错误，请在重要操作前自行确认。
- 自动回复会影响真实聊天体验，建议先小范围测试。
- 发送消息、发送文件、发送名片都是真实操作。
- 查询微信数据只应服务于你自己的使用需求。
- 外部工具服务由对应服务方提供，稳定性取决于服务本身。
- 如果结果很多，优先让 AI 导出文件，不要一次性展示全部内容。
- 从 MCP 市场添加外部服务时，请确认服务来源、权限和 token 使用范围。

## 常见问题

### 为什么 AI 有时候没有回复？

可能原因：

- 自动回复没有开启
- 当前会话被单独关闭
- 群聊消息没有 @ 你
- AI 模型配置不完整
- 外部工具连接失败
- 正在处理上一条消息

### 为什么群聊只在 @ 我时回复？

这是为了避免 AI 在群里频繁插话。群聊默认只处理 @ 你的消息。

### 为什么查询结果没有完全展示？

大量数据会影响阅读和性能。你可以让 AI 导出文件，或者按分页、时间范围继续查询。

如果是工具调用参数、工具结果或最终回复被折叠，点击对应卡片即可展开。长按可以复制完整内容。

### 为什么发送文件需要本地路径？

AI 需要知道文件在手机上的位置，才能把它发送出去。你可以先让 AI 生成文件，再让它把生成的文件发给指定联系人。

### 外部工具连接失败怎么办？

进入外部工具配置页，检查服务地址是否有效、是否启用、是否过期。如果地址过期，需要换新的地址。

如果不知道去哪找新的 MCP 服务，可以在外部工具配置页点击“MCP 市场”，进入 ModelScope MCP 市场查找可用服务。

### 为什么我新增了 MCP 服务，但 Agent 没有使用？

可能原因：

- MCP 服务没有启用。
- 服务地址、连接方式或 token 不正确。
- Agent 已经打开，需要重新进入微信 Agent 才会重新加载工具。
- 当前问题不需要调用该 MCP 工具。
- MCP 服务连接失败，顶部状态或消息列表里会显示连接失败原因。
